LSST Observing Strategy Workshop

Bremerton, WA

August 20-22, 2015

  • Tell Me More

What do you think the LSST observing strategy should be?

This workshop is the second one in a series of LSST workshops aimed at engaging the community interested in LSST to help optimize its observing strategy. There will be several more such "cadence workshops" prior to first light, and the science-based optimization of the survey strategy is likely to continue into at least the commissioning phase.

Workshop Goals:

1) Engage a broad community in defining more robust and quantitative science drivers for open areas (the coverage of the Galactic Plane, LMC/SMC, south Galactic Pole area, and the Ecliptic plane; modifications of uniform temporal sampling, etc.).

2) Begin the definition and editing of a community white paper about LSST observing strategy and its science-driven optimization.

3) Present a status update on cadence optimization, and report on the analysis of the existing set of about a dozen simulated surveys.

4) Describe tools developed by the LSST Project for assessing the scientific potential of simulated cadences (Metrics Analysis Framework), and provide expert training for those who are willing to adopt the MAF in their research and LSST-related work. There will be pre-workshop activities describing and introducing MAF. Participation in these activities, or familiarity with MAF, is not required to attend the workshop.

The target audience for this meeting is everyone willing to contribute to a quantitative definition of the LSST observing strategy through development of specific science programs consistent with the overall LSST mission and the LSST Science Requirements Document. The attendees will include people with MAF experience who want to spend quality time with MAF developers, and work on MAF implementations of science-driven metrics.

Schedule

Thursday, Aug 20, 2015

Morning:
9:00-10:30am, Data Management Update for Scientists (J. Kantor and DM Team).
11:00-12:30am, LSST Overview for Newcomers (Z. Ivezic)

Afternoon: Plenary presentations about LSST observing strategy. (These will include: a status report, plans and goals for the workshop, and an introduction to a white paper about the LSST observing strategy.)

Evening (7:30-9:00) The Project and Community Town Hall Meeting: “Everything you always wanted to know about LSST but were afraid to ask”

Friday, Aug 21, 2015

Morning: Group work, organized by science topics, including (but not limited to): the Galactic Plane (both static and time-domain programs), LMC/SMC, south Galactic Pole area, the Ecliptic plane, modifications of uniform temporal sampling, etc..

Afternoon: Several parallel tracks, according to interests and technical background:
a) General discussions, on topics including: funding for science preparatory work, follow-up planning (including transient broker discussions), cross-facility work, etc.
b) Work on the white paper on LSST observing strategy: discussions of goals and timeline, editorial assignments, etc.
c) Hacking: discussions of detailed quantitative observing strategy optimization, including either pseudo-coding or MAF coding (supported by MAF experts).

Evening: Unconference: several parallel sessions whose topics will have been determined during the day by the workshop participants.

Saturday, Aug 22, 2015

Morning: Plenary debriefs by cadence group leaders, and planning for work beyond the workshop. Workshop ends with lunch.

Installation instructions and MAF tutorials

Installation of the LSST framework

If you already have a working copy of the LSST software you can use the install instructions described below (which should be substantially quicker) or you can update your current software to the latest version by running

eups distrib install sims_maf -t sims

For other installations, the LSST software can now be installed from binaries using conda (the package management system used by Anaconda). The basic steps for installation are as follows:

If conda is not already installed, in a new terminal window download and install miniconda using the instructions given here (install Python 2.7). Note that the conda installation requires that you use bash for the initial installation. At the end of your installation check that the bin directory for conda is in your path by looking at the output from "printenv PATH"

Configure conda so that it knows where to access the LSST files using,

conda config --add channels http://eupsforge.net/conda/dev

Install the LSST Metrics Analysis Framework

conda install lsst-sims-maf

The install requires about 1GB of space and should take approximately 15-30 minutes (depending on your network)

To be able to run the Metrics Analysis Framework we need to setup up some environmental variables so the system knows where the code is installed

[for bash shell users] source eups-setups.sh
[for (t)csh users] source [anaconda directory]/bin/eups-setups.csh
setup sims_maf

Alternate installation of the LSST framework using a virtual machine

If you run into difficulties installing the precompiled version of the Metrics Analysis Framework we recommend that you install it in a Docker virtual machine (see here for instructions for installing Docker if you are using a Mac and pointers to instructions for other operating systems). Once Docker is installed (these instructions are for a Mac but may work for other systems):

docker run -i -e USER=maf -v /Users/XXX/YYY:/home/maf/YYY -P -p 8888:8888 --name maf -t lsst/maf

This downloads MAF and runs it. It mounts the directory /Users/XXX/YYY (which you should change to the directory where you will do your work with MAF).

This will start an interactive bash shell in the directory /home/maf. You will see your directory /Users/XXX/YYY (e.g. the directory you wish to write your notebooks in) as YYY within the docker container.

To setup the Metrics Analysis Framework

source startup.sh

How to run the Metrics Analysis Framework

Checkout the example ipython notebooks and tutorials in the terminal that you are running conda or within the Docker container (Note: checkout these notebooks into the 'YYY' directory in the Docker container as this will also make them visible outside of Docker). You may need to check that "git" is installed on your system.

git clone https://github.com/LSST-nonproject/sims_maf_contrib.git

For the conda installation

ipython notebook sims_maf_contrib/tutorials/Introduction\ Notebook.ipynb

This should open an ipython notebook in a browser. You can follow the instructions in the notebook by executing each cell (with command-enter or control-enter). For an introduction to iPython notebooks see here

For the Docker installation

ipython notebook --ip=0.0.0.0 --no-browser sims_maf_contrib/tutorials/Introduction\ Notebook.ipynb

Check the ip address for the docker container (this assumes a MacOSX installation)

boot2docker ip

Open your browser and point to the ip address and port http://[ip from above]:8888 This should open an ipython notebook in a browser. You can follow the instructions in the notebook by executing each cell (with command-enter or control-enter). For an introduction to iPython notebooks see here

If you exit the container at any point you can reenter it with

docker start -i maf
